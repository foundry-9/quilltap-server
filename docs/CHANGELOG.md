# Quilltap Changelog

## Recent Changes

### 3.3-dev

- feat: **Help Chat practical titling** — help chats now get their own title generation system with plain, descriptive prompts instead of the literary/poetic titles used for Salon chats; titles fire immediately after the first Q&A exchange (interchange 1) instead of waiting until interchange 2; separate prompts for initial titling, title consideration, and title-from-summary all prioritize specificity and findability (e.g., "Setting up Anthropic API connection" rather than "Whispers of Configuration"); manual regenerate-title action also uses help-specific prompts; adds unit tests for help chat interchange-1 behavior
- feat: **Help Chat suggested links and entity picker** — help search results now automatically generate "Related pages" navigation links at the bottom of the chat window, extracted directly from search result URLs rather than relying on the LLM to call `help_navigate`; links are sorted by search relevance and deduplicated against any explicit navigation links the LLM provides; parameterized help URLs (e.g., `/aurora/:id/edit`) now show an entity picker overlay instead of navigating to a broken literal `:id` path — the picker fetches the appropriate entity list (characters, chats, or projects) from the API and lets the user select one before navigating; new `qt-help-suggested-*` and `qt-help-entity-picker-*` CSS classes with theme-overridable custom properties; `qt-floating-dialog-body` gains `position: relative` to scope picker overlays
- fix: **Help Chat tool execution UX** — two fixes: (1) no busy indicator while tools are running — the Help Chat now shows "Consulting the archives..." with the character's avatar during agent-mode tool execution instead of leaving stale text or a blank state; adds `isExecutingTools` state to the streaming hook, triggered by status events and cleared when new content arrives; (2) intermediate "Let me search..." messages accumulate across agent turns — status events now reset `streamingContent` so only the final response is displayed
- fix: **Text-based tool call parser** — add support for bare `<invoke name="...">` tags (Kimi K2/Moonshot-style) that are emitted without a `<function_calls>` wrapper; new `parseInvokeFormat()` parser runs after `parseFunctionCallsFormat()` to avoid double-matching wrapped invokes; ordering in `stripAllXMLToolMarkers()` also ensures `<function_calls>` blocks are stripped first to prevent gutting content while leaving empty wrappers; add `help_search` and `help_navigate` to tool name aliases and `convertToToolCallRequest()`; bump `@quilltap/plugin-utils` to v1.5.3
