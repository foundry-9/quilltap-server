# Tag Customization

> **[Open this page in Quilltap](/foundry/calliope)**

The Tags customization tab in your Settings is where you control how tags look and function throughout Quilltap. Here you can customize every aspect of your tags' appearance and set up which tags are available for quick-hide.

## Accessing Tag Customization

### Navigate to Settings

1. Click the **Settings icon** in the sidebar footer
2. Look for the **Tags** tab in the Settings menu
3. You'll see all your tags listed with their current customization

If you don't have any tags yet, the list will be empty. Tags are created automatically when you add them to characters or profiles.

## The Tags List

### What You See

The main Tags view shows:

- **Tag name** — The text label of each tag
- **Preview** — A visual preview of how the tag appears
- **Quick-hide status** — Whether the tag is available for quick-hide
- **Edit button** — Click to customize the tag's appearance

### Selecting a Tag

Click on any tag or its preview to open the customization panel where you can adjust its appearance.

## Customizing Tag Appearance

When you open a tag for customization, you'll see several options to make it look exactly how you want:

### Color Selection

#### Background Color

- Click the background color box to open a color picker
- Choose any color for the tag's background
- The color picker shows a palette of common colors
- You can also enter a custom hex color code for precise color control

#### Text Color (Foreground Color)

- Click the text color box to choose what color the tag's text will be
- Ensure good contrast between background and text for readability
- Light backgrounds typically pair well with dark text
- Dark backgrounds typically pair well with light text

### Live Preview

As you adjust colors, you'll see the tag update in real-time in the preview area. This helps you choose colors that look good together before saving.

### Emoji Selection

#### Add an Emoji

- Click the emoji picker to select an emoji
- Choose any emoji that represents your tag
- Examples: 🎭 for "drama", 🐉 for "fantasy", 💭 for "thoughts"
- Emojis add visual distinction and make tags easier to scan

#### Emoji-Only Mode

- Enable this toggle to show only the emoji without the tag name
- Useful for saving space when you have many tags
- The full tag name appears as a tooltip when you hover over it
- Great for keeping your sidebar compact

### Text Styling

Apply formatting to the tag's text:

- **Bold** — Make the tag name thicker and more prominent
- **Italic** — Slant the tag name for a stylized look
- **Strikethrough** — Cross through the tag name (useful for "archived" tags)

You can combine multiple styles (e.g., bold + italic).

## Quick-Hide Configuration

At the bottom of each tag's customization panel, you'll find the **Quick Hide** option:

### Enable Quick-Hide

Toggle the **Quick Hide** switch to enable this tag for quick-hide:

- **Enabled** (ON) — The tag appears in the quick-hide menu in the sidebar
- **Disabled** (OFF) — The tag won't appear in quick-hide (it's still usable, just not for hiding)

### Which Tags to Mark for Quick-Hide

Best practice is to mark only your high-level organizational tags for quick-hide:

- **Yes for quick-hide** — "archived", "inactive", "wip", "project-a"
- **No for quick-hide** — Detailed tags like "romance", "dialogue", "needs-review"

This keeps the quick-hide menu focused and organized.

## Saving Changes

All customization changes are saved automatically as you make them:

- No "Save" button needed
- Changes appear across Quilltap immediately
- Tags update in all locations (sidebar, character profiles, search results, etc.)

## Design Best Practices

### Color Selection Tips

- **Use consistent color scheme** — Choose colors that work well together
- **Ensure readability** — Check that text is readable against the background
- **Consider dark mode** — Test your colors in both light and dark modes
- **Limit palette** — Use 5-10 main colors to keep tags visually cohesive
- **Semantic colors** — Use green for "active", red for "archived", blue for "wip"

### Emoji Tips

- **Be consistent** — Use emojis that make sense for your system
- **Match the tag meaning** — 🎭 for drama, 🔥 for popular, 📚 for story
- **Size matters** — Emojis display at different sizes; test a few to see what you like
- **Accessibility** — Don't rely on emoji alone; keep the tag name for clarity

### Text Styling Tips

- **Bold for important** — Mark frequently-used or important tags in bold
- **Italic for style** — Use sparingly for special tags
- **Strikethrough for inactive** — Mark archived content with strikethrough
- **Consistency** — Apply the same styling to related tags

## Renaming Tags

Tags are named when you create them. To change a tag's name:

1. The tag name is set when first created
2. Currently, tag names can't be edited in the customization interface
3. To rename a tag, you would need to recreate it with the new name
4. Then update all tagged content to use the new tag

Consider your tag naming carefully before creating many tags!

## Tag Management

Below the Tag Appearance section, you'll find the **Tag Management** section which lists every tag in your workspace.

### Viewing Tag Usage

Each tag row shows:

- **Tag preview** — How the tag currently looks with its styling
- **Usage count** — How many entities (characters, chats, profiles, files) use this tag
- **Delete button** — Remove the tag entirely

### Deleting Tags

To delete a tag:

1. Find the tag in the **Tag Management** list
2. Click the **Delete** button on the tag's row
3. A confirmation popover will appear showing how many items use the tag
4. Click **Delete** to confirm, or **Cancel** to back out

When you delete a tag:

- The tag is removed from **all** entities that use it (characters, chats, connection profiles, image profiles, embedding profiles, and files)
- The tag's visual style and quick-hide settings are permanently removed
- This action cannot be undone

Unused tags (showing "Unused") can be safely deleted without affecting any content.

## Organizing Your Tag System

### Before You Start

Think about your needs:

- What are your main categories? (genres, projects, status, etc.)
- How many tags do you need? (aim for 10-20 main tags)
- What naming convention? (lowercase, underscores, specific prefixes)
- What's your quick-hide strategy? (which tags are for broad hiding?)

### Creating Your System

1. **Define categories** — Decide what your tags represent
2. **Name consistently** — Use a consistent naming convention
3. **Customize appearance** — Make tags visually distinct by category
4. **Set quick-hide** — Mark your high-level organizational tags for quick-hide
5. **Document** — Remember what each tag means

### Maintaining Your System

- **Review monthly** — Are your tags still relevant?
- **Consolidate** — Merge similar tags if you have too many
- **Expand** — Add new tags as your needs evolve
- **Clean up** — Remove tags you're no longer using

## Color Picker Guide

### Quick Colors

The color picker provides quick access to common colors:

- Reds, pinks, purples
- Blues, teals, greens
- Yellows, oranges
- Grays, blacks, whites

### Custom Colors

For precise control:

1. Click the hex input field
2. Enter a color code (e.g., #FF5733)
3. Or use the color wheel to pick any color
4. Click outside the picker or press Enter to apply

### Testing Colors

- Check colors against both light and dark themes
- Ensure text is readable
- Consider color-blind accessibility
- Test on your actual character tags to see them in context

## Troubleshooting

### Changes Not Showing

- Refresh your browser if changes aren't immediate
- Log out and log back in
- Clear your browser cache

### Can't Find a Tag

- Ensure you've tagged at least one item with the tag
- Check the correct Settings section
- Search for the tag to verify it exists

### Colors Look Different Elsewhere

- Different devices/screens display colors differently
- Light/dark mode affects color appearance
- Test on the devices you use most

The Tags customization system is flexible—experiment to find a setup that works perfectly for your workflow!
