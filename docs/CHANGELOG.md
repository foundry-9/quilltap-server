# Quilltap Changelog

## Recent Changes

### 3.1.2

- fix: Remove dead mount-points references from legacy JSONL files migration ‚Äî the INSERT used `mountPointId`, `s3Key`, and `s3Bucket` columns that no longer exist on fresh databases (mount-points system was removed in v2.9), causing first-startup migration failure

### 3.1.1

- fix: npm `quilltap` package now publishes with the correct release version ‚Äî the release workflow was publishing whatever version was already in `packages/quilltap/package.json` (e.g., `3.1.0-dev.36`) instead of the git tag version (e.g., `3.1.0`), causing the CLI to look for a non-existent GitHub Release asset; added `npm version` step to sync package version to the release tag before publishing

### 3.1.0

- chore: Remove dead debug logging artifacts ‚Äî delete unused `requestPayload` object in streaming service and stray `console.debug` in ProviderWizard component
- docs: Comprehensive documentation audit ‚Äî fixed inaccuracies across 60 files; replaced all stale `/foundry/*` UI paths with `/settings?tab=*` in 28 help files and 6 docs files; added `/v1/` prefix to ~80+ API endpoint references across 10 docs; replaced MongoDB/Prisma/S3 references with SQLite/Zod/local-filesystem in 5 feature and architecture docs; rewrote migrations README table from 14 entries to all 46; expanded PROMPT_ARCHITECTURE.md from 9 lines to full architectural reference; updated chat-settings README file listing (7‚Üí16 files); updated .githooks README to document all 12 pre-commit steps; fixed plugin manifest capability count (20‚Üí22); corrected theme count and names in README; updated CLAUDE.md self-references; rebuilt help bundle
- refactor: Move AI Character Import wizard from Settings ‚Üí Templates to Aurora page ‚Äî now accessible via "Summon From Lore" button alongside "Import from SillyTavern" and "Create Character"; opens as a large dialog overlay; wizard accepts `onClose` and `onImportSuccess` callbacks so character list refreshes after import; renamed SillyTavern import button and dialog title for clarity; updated help documentation
- feat: Enable Node.js (npx) runtime mode on Linux ‚Äî Linux Electron users can now choose between Docker and Node.js runtimes in the splash screen; previously Linux was locked to Docker-only with the VM and Node.js buttons hidden; the settings loader now persists the `'npx'` choice on Linux instead of forcing `'docker'`; fallback logic is platform-aware so Linux falls between Docker and Node.js instead of to the nonexistent VM mode; adds `platform` field to `DirectoryInfo` for clean platform detection in the splash UI
- feat: Node.js (npx) runtime mode in Electron app ‚Äî adds a third "Node.js" button to the splash screen alongside Docker and VM, allowing users with Node.js >= 18 installed to run the backend directly via `npx quilltap@{version}` without needing Docker or a VM; new `NpxManager` class handles process lifecycle (spawn, health check, graceful SIGTERM/SIGKILL shutdown, Windows `taskkill` tree cleanup); probes well-known Node.js install paths (homebrew, nvm, fnm, system) since packaged Electron has minimal PATH; splash screen resized to 3-column button layout; all startup sequences stop conflicting backends before launching to prevent port conflicts
- fix: Native module resolution for `npx quilltap` ‚Äî native modules (`better-sqlite3`, `sharp`, `@img/sharp-*`) are now symlinked into the standalone directory's `node_modules/` so standard Node.js resolution finds them without relying on `NODE_PATH`; `ensureNativeModules()` uses `require.resolve()` to locate modules regardless of npm hoisting; `NODE_PATH` fallback now includes the parent `node_modules/` to handle hoisted dependencies; fixes "Cannot find module 'better-sqlite3'" error when running via `npx`
- fix: Native module version check now loads the `.node` binary directly ‚Äî `better-sqlite3` lazy-loads its native binding only when a Database is created, so the previous `require('better-sqlite3')` check always succeeded without testing the actual binary; the CLI now loads the compiled `better_sqlite3.node` file directly from the package's `build/Release/` directory to reliably detect NODE_MODULE_VERSION mismatches
- fix: BLOB column support in SQLite update path ‚Äî `translateUpdate`/`buildUpdateQuery` now accept blob columns so that `updateOne`/`updateMany` write embeddings as Float32 BLOBs instead of JSON TEXT; previously the normalize-vector-storage migration's work was undone by subsequent writes; threads `blobColumns` from `SQLiteCollection` through to the query translator
- fix: Memory cleanup POST returned 400 ‚Äî the housekeeping dialog was not including `characterId` in the POST body despite the API schema requiring it; added missing field to the fetch body in `housekeeping-dialog.tsx`
- perf: Memory cleanup merge pass no longer makes N embedding API calls ‚Äî rewrote the "merge similar memories" pass in housekeeping to use already-stored embeddings from the vector store instead of calling `findSimilarMemories()` per memory (which generated a new embedding via external API and fetched all memories from the DB on each iteration); preview with 125+ memories is now essentially instant
- fix: Buffer hydration fallback in SQLiteCollection ‚Äî when a BLOB value arrives in a column not registered in `blobColumns` (timing/registration edge case), `hydrateRow` now decodes it as Float32 instead of storing the raw Buffer; prevents `Buffer.length` (byte count) from being mistaken for embedding dimension count (float count), which caused "Vectors must have the same length" errors
- fix: Vector store search returns empty on dimension mismatch instead of throwing ‚Äî `CharacterVectorStore.search()` now logs a debug message and returns `[]` when query dimensions don't match, preventing transient mismatches during migration from crashing the entire operation; per-entry dimension check also skips mismatched entries gracefully
- fix: New migration `fix-text-embeddings-after-update-v1` ‚Äî converts any TEXT embeddings in `memories` and `vector_entries` back to Float32 BLOBs; cleans up damage caused by the buggy update path that wrote embeddings as JSON TEXT after the normalize-vector-storage migration had already converted them
- fix: Auto-rebuild native modules when Node.js version changes ‚Äî the `quilltap` CLI now detects `NODE_MODULE_VERSION` mismatches (e.g., after upgrading Node.js while npx has a cached install) and automatically runs `npm rebuild` for `better-sqlite3` and `sharp` before starting the server; prevents the "was compiled against a different Node.js version" crash on startup
- refactor: Rename Dangermouse subsystem to the Concierge ‚Äî updates subsystem ID, display name, image paths, UI strings, plugin manifests, documentation, and all code comments; no functional changes
- feat: Global search now includes message content ‚Äî the Cmd+K search dialog searches within chat message text in addition to chat titles, character names, tags, and memories; results show the chat name, a "You"/"Character" role badge, character names, and a highlighted snippet of the matching message; adds "Messages" filter chip, `searchMessagesGlobal()` database method, amber `qt-badge-message` CSS token, and `MessageSearchResult` type
- feat: Add MODERATION_PROVIDER plugin capability with OpenAI implementation ‚Äî the Concierge now automatically uses OpenAI's free moderation endpoint (`/v1/moderations`) for content classification when an OpenAI connection profile is configured; adds new `MODERATION_PROVIDER` plugin capability, `ModerationProviderPlugin` interface in `@quilltap/plugin-types`, moderation provider registry, and OpenAI moderation provider in `qtap-plugin-openai`; falls back transparently to the existing Cheap LLM classification when no moderation provider is available; moderation endpoint is purpose-built, free to use, and returns structured category scores mapped to Concierge categories (sexual‚Üínsfw, hate‚Üíhate_speech, violence‚Üíviolence, self-harm‚Üíself_harm, illicit‚Üíillegal_activity)
- fix: Context summary now regenerates on the same schedule as title and story background checks ‚Äî previously the context summary only regenerated after 100+ messages, leaving chat-level Concierge classification with stale/terse summaries; now force-regenerates at interchange checkpoints (2, 3, 5, 7, 10, then every 10) matching the title reconsideration schedule
- fix: Moderation provider no longer reports irrelevant categories ‚Äî OpenAI returns tiny nonzero scores (e.g. 0.0001) for every category even when content doesn't match; the category mapper now filters with a 1% relevance floor so only meaningfully scored categories appear in classification results
- fix: Deduplicate EMBEDDING_GENERATE background jobs ‚Äî `enqueueEmbeddingGenerate()` now checks for an existing PENDING/PROCESSING job for the same entity before creating a new one, preventing hundreds of duplicate jobs during reindex-all and normal usage; adds `findPendingForEntity()` repository method; follows the same dedup pattern used by story background and danger classification jobs
- fix: Agent mode now inherits from character/project defaults in the UI ‚Äî new chats with a character or project that has `defaultAgentModeEnabled: true` now correctly show "Agent On" in the tool palette; the chat GET endpoint and toggle action now resolve the full cascade (global ‚Üí character ‚Üí project ‚Üí chat) and return `resolvedAgentModeEnabled` alongside the raw chat-level value; frontend reads the resolved value for display
- ci: Add `notify-website` job to release workflow ‚Äî triggers a `repository_dispatch` event to `foundry-9/quilltap-website` after GitHub Release is created, passing version, prerelease flag, and Docker image tag; uses `peter-evans/repository-dispatch@v3` with a fine-grained `CROSS_REPO_PAT`; exposes `prerelease` output from `create-release` job for downstream consumption
- fix: WSL2 MCP localhost rewrite ‚Äî MCP servers on the Windows host were unreachable from WSL2 because the `/proc/net/route` gateway IP doesn't forward to `127.0.0.1`-bound services; added WSL2-specific strategy that reads the nameserver from `/etc/resolv.conf` (which WSL2 auto-generates to point at the Windows host with special localhost forwarding); detects WSL2 via `/proc/sys/fs/binfmt_misc/WSLInterop`; mirrors fix in `@quilltap/plugin-utils`
- fix: File browser now correctly places files in their subfolder instead of showing all files at root ‚Äî files with empty `folderPath` in the database are now resolved using their `storageKey` physical path; adds `deriveFolderPathFromStorageKey()` and `resolveEffectiveFolderPath()` utilities to `folder-utils`; reconciliation now detects and corrects `folderPath` mismatches on matched files; watcher uses shared utility for consistent path derivation; filesystem sync fixes existing data permanently
- fix: File browser folder file counts now show the correct number of contained files instead of always displaying "1 file" ‚Äî derived folder counting logic had a dead code path that prevented incrementing the count beyond the first file
- fix: Physical database backups now run once per day instead of on every startup ‚Äî checks the most recent backup timestamp and skips if less than 24 hours old; prevents excessive backups during development (HMR restarts) and frequent production restarts; applies to both main database and LLM logs database backups
- fix: Watcher and reconciliation now handle physical file moves ‚Äî when a file is moved on disk (e.g., drag in Finder), the watcher defers unlink events for 3 seconds and cross-matches by SHA-256 when the corresponding add event arrives, preserving all tags, links, and metadata; startup reconciliation similarly cross-matches unmatched disk files against unmatched DB records by SHA-256 before creating orphaned records or deleting stale ones
- feat: Filesystem-backed file storage restructuring ‚Äî files are now stored in real directories on disk using their original filenames (no userId prefix, no fileId prefix); new storage key format `{projectId}/_general/{folderPath}/{safeFilename}`; `.meta.json` sidecar files eliminated in favor of database-only metadata; chokidar filesystem watcher detects added/removed/changed files in real-time; startup reconciliation syncs DB with filesystem; one-time migration moves existing files to new layout; new `Sync Now` button in file browser triggers manual reconciliation; untracked files (found on disk without DB records) shown with amber indicator; backup format v2 stores files by storageKey path with backward-compatible restore for old format; new scanner module for directory traversal and SHA-256 hashing; updated help documentation
